Tables or divs?

A few years ago the World Wide Web Consortium (W3C) decided that tables can no longer be used for layout purposes, only for data. Tables offer a lot more information to a user than a div would, so in the case of layouts there is no need to have content inside a table that is only with a certain layout to look nice.

Complex tables and captions

Since not all visitors to your website can understand the layout of tables and the associations between table headers and their respective cells, screen readers are built to recognize these features. Below you will find some information on how this can be achieved successfully.

  1. Create your table using the table tag
  2. Add a <caption> explaining the information in your table. This can be used to add a title to your table. This information will be used by screen readers.
  3. Add a <thead> and </thead> where you would add your headers: <head>.
  4. Add a <tbody> and </tbody> and add your content using <tr> and <td>.
